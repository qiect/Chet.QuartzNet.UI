# 🔥 Chet.QuartzNet.UI 新增作业指南：3分钟快速创建可视化定时任务！（todo）

## 🎯 什么是新增作业功能？

新增作业功能是 Chet.QuartzNet.UI 提供的核心功能之一，它允许你通过可视化界面快速创建 Quartz.Net 作业，无需编写复杂的配置代码。无论你是新手还是资深开发者，都能轻松上手！😍

## ✨ 新增作业有哪些类型？

Chet.QuartzNet.UI 支持多种作业类型，满足不同场景的需求：

### 🔧 基本作业 (Basic Job)
- 最常用的作业类型
- 支持定时执行单个任务
- 配置简单，适合大多数场景

### 📦 分组作业 (Group Job)
- 将多个作业组织到一个组中
- 方便管理和分类
- 支持按组查看和筛选

### 🎯 自定义作业 (Custom Job)
- 支持自定义作业逻辑
- 可以调用外部服务或执行复杂业务流程
- 灵活性最高，适合特殊需求

### 📅 日历作业 (Calendar Job)
- 支持基于日历的调度
- 可以设置排除特定日期（如节假日）
- 适合需要考虑工作日的场景

## 🚀 如何新增一个作业？

### 1️⃣ 登录管理界面

访问 `http://localhost:port/quartz-ui`，使用用户名和密码登录。

### 2️⃣ 进入作业管理页面

点击左侧菜单的「作业管理」，然后点击右上角的「新增作业」按钮。

### 3️⃣ 填写作业基本信息

| 字段名 | 说明 | 示例 |
|-------|------|------|
| 作业名称 | 作业的唯一标识符 | SampleJob |
| 作业组 | 作业所属的组，用于分类管理 | DEFAULT |
| 作业描述 | 作业的简要说明 | 示例作业 |
| 作业类型 | 选择作业的类型 | 基本作业 |

### 4️⃣ 配置调度表达式

Chet.QuartzNet.UI 支持多种调度表达式类型：

#### 🔄 Cron 表达式
- 最灵活的调度方式
- 支持复杂的时间规则
- 提供 Cron 表达式生成器，无需手动编写

示例：每5分钟执行一次
```cron
0 0/5 * * * ?
```

#### ⏰ 简单调度
- 适合简单的定时任务
- 可以设置重复间隔和执行次数

示例：延迟10秒执行，每30秒重复一次，共执行5次

#### 📅 日历调度
- 基于日历的调度方式
- 可以选择工作日或排除特定日期

### 5️⃣ 配置作业参数

你可以为作业添加自定义参数，作业执行时可以获取这些参数：

| 参数名 | 参数值 |
|-------|--------|
| Param1 | Value1 |
| Param2 | Value2 |

### 6️⃣ 配置通知设置

选择作业执行成功或失败时是否发送通知：
- 发送成功通知
- 发送失败通知
- 选择通知模板

### 7️⃣ 保存并启用作业

点击「保存」按钮，作业创建完成。你可以选择是否立即启用作业。

## 💡 新增作业的最佳实践

### 🎯 作业命名规范
- 使用有意义的作业名称，便于识别
- 作业组名使用业务领域相关的名称
- 避免使用特殊字符

### ⏰ 调度表达式优化
- 避免在系统高负载时间段执行大量作业
- 合理设置作业间隔，避免资源竞争
- 对于长时间运行的作业，考虑使用异步执行

### 📊 参数配置技巧
- 只传递必要的参数
- 参数值尽量简洁
- 对于敏感信息，考虑使用加密存储

### 🔔 通知设置建议
- 对于关键作业，同时启用成功和失败通知
- 对于非关键作业，只启用失败通知
- 合理设置通知频率，避免通知泛滥
